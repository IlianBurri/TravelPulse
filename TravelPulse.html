M122 Projektarbeit · Bash-Skript

Reise-Infos,
ein Befehl.

TravelPulse holt Ortszeit, Wetter, Wechselkurs und Krypto-Kurse für jede Stadt der Welt — und fragt auf Wunsch ein lokales KI-Modell nach Reisetipps. Alles direkt im Terminal, ohne API-Keys.

Repository ansehen Installation
bash — travelpulse.sh 

      

    

  


  
  

    

      

        
An Bord

        
Fünf Datenquellen, eine Abfrage

        
Jede Abfahrtstafel zeigt nur, was gerade relevant ist. TravelPulse macht
           dasselbe für deine Reise — kompakt, aktuell, ohne Umwege.

      


      

        

          Nr.DatenQuelleStatus
        

        

          01
          Ortszeit
          Aktuelle Uhrzeit der Zielstadt via Zeitzonen-API
          Aktiv
        

        

          02
          Wetter
          Temperatur, Wind & Zustand via Open-Meteo
          Aktiv
        

        

          03
          Wechselkurs
          Heimwährung → Landeswährung via Frankfurter API
          Aktiv
        

        

          04
          Krypto-Kurse
          BTC & ETH in USD via CoinPaprika
          Aktiv
        

        

          05
          KI-Assistent
          Lokales Modell via Ollama beantwortet Reisefragen
          Optional
        

      

    

  


  
  

    

      

        
Ablauf

        
Vom Tippen bis zur Antwort

        
Jede Eingabe wird validiert, jede Anfrage abgesichert. So sieht der
           Weg vom ersten Befehl bis zur fertigen Übersicht aus.

      


      

        

          
1

          

            
Stadt & Heimwährung eingeben

            
Bei leerer Eingabe fragt prompt_required() erneut —
               das Skript bricht nicht grundlos ab.

          

        

        

          
2

          

            
Stadt via Geocoding auflösen

            
Open-Meteo liefert Land, Zeitzone und Koordinaten. Unbekannte
               Städte führen zu einer klaren Fehlermeldung.

          

        

        

          
3

          

            
Vier Abfragen parallel

            
Zeitzonen-API, Wetter, Wechselkurs und Krypto-Kurse werden
               unabhängig abgefragt — fetch_json() wiederholt
               fehlgeschlagene Calls bis zu dreimal.

          

        

        

          
4

          

            
Übersicht ausgeben

            
Alle Werte werden formatiert und gruppiert im Terminal
               dargestellt — Zeit, Standort, Wetter, Wechselkurs, Krypto.

          

        

        

          
5

          

            
Optional: KI-Assistent starten

            
Ist Ollama unter localhost:11434 erreichbar, zeigt
               TravelPulse installierte Modelle an und startet einen Chat
               zur gewählten Stadt.

          

        

      

    

  


  
  

    

      

        
Einstieg

        
In zwei Befehlen startklar

        
Benötigt curl und jq. Der KI-Assistent ist
           optional und nutzt eine lokale Ollama-Instanz.

      


      

        

          
Repository einrichten

          
          
# Repo klonen & ausführbar machen
git clone https://github.com/IlianBurri/TravelPulse.git
cd TravelPulse
chmod +x travelpulse.sh

# starten
./travelpulse.sh

        


        

          
Abhängigkeiten (Ubuntu / WSL2)

          
          
# curl & jq installieren
sudo apt update && sudo apt install curl jq

# optional: lokaler KI-Assistent
ollama pull llama3.2
